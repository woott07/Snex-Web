Overview

Snax is a highly modular, multi-purpose Discord bot built with discord.js v14. It combines a full-featured music player, a rich permission/role system, server moderation tools, voice management utilities, and smart anti-spam protection — all in one lightweight, easy-to-extend package.

Default prefix: $ — customisable per-server with $setprefix. You can also trigger commands by @mentioning the bot instead of using the prefix.

Setup & Installation

Prerequisites

  • Node.js v18 or higher — nodejs.org
  • Discord Bot Application & token — Discord Developer Portal
  • YouTube Data API v3 key — Google Cloud Console
  • FFmpeg — bundled via ffmpeg-static, no manual install needed.
  • yt-dlp — must be installed and accessible in your system PATH.

Windows

  1. Double-click install.bat. This runs npm install and creates your .env and .gitignore automatically.
  2. Open .env in any text editor and fill in your credentials (see Configuration section).
  3. Run the bot: node index.js

macOS / Linux

  1. Run the setup script: bash setup_mac.sh
  2. Fill in .env (see Configuration section).
  3. Run the bot: node index.js

Configuration

.env File (Secrets — Never Share)

Bot_Token=YOUR_DISCORD_BOT_TOKEN
YOUTUBE_API_KEY=YOUR_YOUTUBE_DATA_API_V3_KEY
  • Bot_Token — Your Discord bot token from the Developer Portal.
  • YOUTUBE_API_KEY — Google Cloud YouTube Data API v3 key. Required for text-search. Direct YouTube URLs still work without it.

config/config.json (Runtime Settings)

{
  "prefix": "$",                   // Default command prefix for all servers.
  "defaultVolume": 100,            // Starting playback volume (0–100).
  "leaveOnEmpty": true,            // Leave voice channel when it becomes empty.
  "leaveOnEmptyCooldown": 30000,   // Milliseconds to wait before leaving (30 s).
  "leaveOnEnd": false,             // Stay in VC after the queue finishes.
  "autoJoin": false,               // Auto-join on message (not implemented yet).
  "selfDeaf": true,                // Bot deafens itself while in VC.
  "embed": {
    "color": "#2b2d31",            // Hex colour for all bot embeds.
    "showThumbnail": true,         // Show song thumbnail in Now Playing embed.
    "showRequester": true          // Show who requested the song.
  },
  "maxQueueDisplay": 10,           // Max songs shown per queue page.
  "permissions": {
    "sameVoiceRequired": true,     // Users must be in the same VC to control music.
    "djRole": "",                  // Role name/ID for a dedicated DJ role (unused).
    "adminBypass": true            // Admins bypass all voice checks.
  }
}

Per-server prefix overrides are stored automatically in data/permissions.json and take precedence over the global prefix in config.json.

Permission System (RBAC)

Snax uses a custom Role-Based Access Control (RBAC) system stored in data/permissions.json. Permissions are evaluated per-server (guild).

Priority Order

  1. Server Owner — unrestricted access to everything.
  2. AdminExe — bypasses all permission checks.
  3. Assigned permission group matching the command.
  4. @everyone (if Default group is set to @everyone).
  5. Denied — command blocked.

Permission Groups

GroupCommands Unlocked
Defaultplay, skip, queue, ping, help, hello, nowplaying, volume, pause, resume, prev, remove, clear, shuffle, loop, autoplay, stop, join, leave
VoiceExemute, unmute, deafen, undeafen, dragreq, addme
setNickExesetnick
ChatExepurge, timeout
ManagerExeban, unban, kick — plus can assign: VoiceExe, setNickExe, assign_default, default_remove
AdminExeAll of the above, plus all admin/assignment commands and prefix management
BypassExeAnti-spam extended limit (15 warns / 20 timeout, vs 4/7 default)
SupBypassTotal anti-spam immunity — no warnings or timeouts ever applied
OwnerOnlyCommands not in any group — Server Owner only

AdminExe 10-Minute Cooldown

When AdminExe is granted with $AdminExe @target, the target is placed in a pendingAdminExe queue with a 10-minute delay before becoming a full AdminExe. This prevents accidental instant privilege escalation. If $remAdminExe is called before the 10 minutes elapse, the pending entry is removed immediately.

Data Storage Format

{
  "<guild_id>": {
    "Default": ["@everyone"],
    "VoiceExe": ["<role_or_user_id>", ...],
    "setNickExe": [...],
    "ChatExe": [...],
    "ManagerExe": [...],
    "AdminExe": [...],
    "pendingAdminExe": { "<id>": <unix_timestamp_ms> },
    "BypassExe": [...],
    "SupBypass": [...],
    "customPrefix": "<prefix>"   // optional — only present when set
  }
}

New guilds are initialised with Default: ["@everyone"] so music commands work for everyone out of the box.

Anti-Spam System

The bot monitors message frequency using an in-memory rolling-window counter. Each user's message timestamps are tracked over a 10-second window.

TierLimit (10s window)Action
Normal user4 messagesWarning sent (auto-deleted after 5s)
Normal user7 messages2-minute Discord timeout applied
BypassExe15 messagesWarning sent
BypassExe20 messages2-minute timeout applied
SupBypassNo limitFully immune — no warnings or timeouts
AdminExeNo limitImmune (AdminExe implies all privileges)
ManagerExeNo limitImmune
Server OwnerNo limitImmune

Only bot commands (prefix / @mention) are checked — regular chat messages are not affected by this filter.

Music System

The music system is an isolated module in the music/ directory and is loaded separately from the general command system.

Audio Extraction Pipeline

  1. User types $play <query>
  2. searchPipeline.js resolves the query — if it's a URL it cleans it (Shorts → watch URL, strips radio-mix params); if it's a text search it calls YouTube Data API v3 and filters out Shorts (<= 60 seconds).
  3. The resolved URL is passed to discord-player using YTDLPExtractor.
  4. YTDLPExtractor calls the native yt-dlp binary to extract a stream URL, bypassing JavaScript cipher decryption issues.
  5. The audio stream is piped through @discordjs/voice to the voice channel.

Queue Embed UI

When a song starts playing, Snax sends an interactive Now Playing embed with the song title, channel/author, duration, and thumbnail. Each queued track has three buttons:

  • [Remove] — Remove that specific song from the queue.
  • [Push to Top] — Move that song to play next (position 1).
  • [Keep] — Acknowledge / dismiss the queue entry.

Buttons on previous embeds auto-expire when a new track is queued, keeping the chat clean.

Adding New Commands

Snax uses a dynamic command loader — adding a command requires only creating a new file. No manual registration is needed.

  1. Create a new .js file in the commands/ directory. For music-related commands, use music/commands/ instead.
  2. Export an object with at minimum: 
    module.exports = {
  name: 'mycommand',             // REQUIRED — command trigger name (lowercase)
  aliases: ['mc', 'mycmd'],      // OPTIONAL — alternative trigger names
  description: 'Does a thing',  // OPTIONAL — shown in help
  async execute(message, args, client, player, config) {
    // Your command logic here
    message.reply('Hello!');
  }
};
  3. Restart the bot. The commandHandler will automatically detect and load the file.
  4. To require a permission group, add the command name to the appropriate group in utils/permissions.js: 
    'VoiceExe': ['mute', 'unmute', ..., 'mycommand'],
    If the command is not listed in any group, it defaults to OwnerOnly.

Known Limitations

YouTube API Quota

The YouTube Data API v3 has a daily quota of 10,000 units per project. Each text-search call uses approximately 102 units (100 for search + 1 per video detail). If the quota is exceeded, the bot notifies the user and refuses to search until the quota resets (midnight Pacific Time). Direct URL playback is NOT affected by API quota.

Shorts Auto-Conversion

YouTube Shorts URLs (youtube.com/shorts/<id>) are automatically converted to standard watch URLs. Shorts are also filtered out of text-search results — any video ≤ 60 seconds is skipped.

Pending AdminExe

AdminExe grants have a mandatory 10-minute delay before taking effect. This is intentional and cannot be bypassed (except by the server owner).

Purge Age Limit

Discord's API does not allow bulk-deleting messages older than 14 days. The $purge command will skip or fail on older messages.

Required Discord Intents

The bot requires these Privileged Intents enabled in the Developer Portal under your application's settings:

  • Message Content Intent
  • Server Members Intent (for role checks)

Local Data Storage

The data/permissions.json file is stored locally on the host machine and is not synced to any external database. Back it up regularly.
